Std.Fiber

The standard structured-concurrency layer over Hiko's isolated runtime processes. Fibers are direct-style and may suspend for real runtime I/O, but they do not share a heap.

Use

import Std.Fiber

Raw source: Fiber.hml

Public surface

  • spawn : (unit -> 'a) -> 'a Fiber.t
  • join : 'a Fiber.t -> 'a
  • cancel : 'a Fiber.t -> unit
  • both : ((unit -> 'a) * (unit -> 'b)) -> ('a * 'b)
  • all : (unit -> 'a) list -> 'a list
  • first : ((unit -> 'a) * (unit -> 'a)) -> 'a
  • any : (unit -> 'a) list -> 'a

Semantics

Std.Fiber is the user-facing layer for real concurrent work. Algebraic effects remain local control-flow tools; runtime scheduling, suspension, and capability checks still belong to the VM/runtime layer.

Current implementation note: join still propagates child failure as a process failure. Typed Result-based joins will come when structured process errors are surfaced to HML.

Example

import Std.Fiber

val response =
  Fiber.first (
    (fn () => http_get "https://example.com/a"),
    (fn () => http_get "https://example.com/b")
  )
